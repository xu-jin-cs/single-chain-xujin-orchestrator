
文档基础信息
文档名称：MCP Xujin 工作流插件需求说明书
版本：V1.0 定稿（防伪 ID 唯一性规则更新）
定位：内部研发落地标准，全部交互、存储、执行、校验、切换规则不可变更
1 产品核心定位与底层约束
1.1 产品定位
面向 Codex、Kimi、Claude 等 MCP 兼容编辑器的流程编排工具；仅负责流程链路、输入输出文件管控、门禁防伪流转，绝不修改、校验、干预任何 Agent 自身身份、技能、业务规则，Agent 内部逻辑完全由用户自定义。
离线运行完全自闭环：MCP 仅作为编辑工具，流程生成后可彻底脱离 MCP 独立运行，无任何运行依赖。
1.2 硬性底层约束
运行层全部配置、数据本地落地，丢失 MCP 服务 / 全局配置文件不阻断离线流程；
MCP 仅管控：文件校验、交付物归档、Agent 门禁哈希防伪、状态分支流转、断点执行；
Agent 身份、技能、业务规则完全开放给用户自定义，MCP 仅存储透传，无内容校验；
Agent 业务配置禁止 JSON / 数组 /{ } [ ]等结构化符号，仅纯文本，防止配置整合解析错乱；
新手 / 高级模式分离，模式状态绑定单工作流flow.yml，全局不统一锁定。
2 部署架构
2.1 双通信运行模式
stdio 模式：单编辑器独占连接；
HTTP 模式：多客户端并发接入编辑；
2.2 离线独立运行机制
编排完成提供 mscp_workflow_disconnect 指令一键脱离 MCP；
离线执行仅依靠终端自定义指令，不读取 MCP 服务、global_config.json；
仅新增 / 修改 / 转换 / 快照流程时需要重连 MCP。
3 全局配置持久化规则
存储载体：MCP 根目录单文件 global_config.json；
存储内容：全局文件黑白名单、全局离线执行指令别名、系统默认新手模式；
隔离机制：
单工作流私有规则存储在自身flow.yml，离线优先生效；
丢失global_config.json，离线流程自动启用内置兜底规则，不崩溃；
global_config.json 仅编辑阶段读取，离线执行器完全不加载。
4 标准工作流目录结构（每个流程独立根目录，名称由用户自定义）
plaintext
自定义工作流名称/
├─ flow.yml                # 主流程自包含配置（离线运行唯一依赖）
├─ templates/              # 交付物模板库
├─ validate_rules/        # 文件校验规则、自定义校验脚本
├─ flow_fragments/        # 分支流转片段
├─ input_source/          # 持久化入参目录（永久留存，无自动清理）
├─ output_delivery/       # 交付物归档，二级目录按Agent名称划分
├─ logs/                  # 分层执行日志，按日期分文件
├─ state_store/           # 单次执行临时防伪ID、Agent状态，新一轮自动清空
└─ versions/              # 简易版本快照目录（无自动清理、无一键清空指令）
4.1 入参持久化规则
input_source 永久留存所有上传输入文件，无自动清理；
每次执行流程前置弹窗提示：「本次输入文件、交付物、日志、临时防伪状态将持久本地存储，不会自动清理，请定期手动清理释放磁盘」。
4.2 交付物统一归档约束
output_delivery/{Agent名称}/ 强制存储所有 Agent 产出，严格匹配配置约定文件名、格式；
流程正常执行完毕，终端打印汇总清单：按 Agent 分组展示交付物名称 + 完整本地路径 + 本轮全局唯一哈希防伪 ID；清单同步写入日志。
4.3 快照规则
生成指令：mscp_workflow_snapshot_create，打包配置 / 模板 / 规则存入versions；
回滚指令：mscp_workflow_snapshot_rollback --snapshot xxx.zip；
快照不含运行产生的 input_source/output_delivery/logs/state_store；
无自动清理、无内置清理指令，快照堆积由用户手动删除文件。
5 文件三层校验 + 双层黑白名单
5.1 黑白名单作用域（全局默认 + 单流程覆盖）
全局黑白名单存储在global_config.json，新建流程默认继承；
单流程flow.yml私有黑白名单离线优先生效；
MCP 支持一键重置单流程规则，恢复继承全局默认配置。
5.2 串行三层校验（任意一层失败直接阻断流程）
基础校验：文件后缀、文件大小阈值黑白名单；
内容轻校验：文本编码、JSON/YAML/PDF/ 图片格式合法性校验；
自定义本地校验脚本钩子，脚本返回失败终止流转。
6 Agent 可视化编辑交互规范（仅 MCP 图形界面支持，终端无表单编辑）
6.1 Agent 自定义业务配置区块（MCP 不校验内容）
每个 Agent 表单内置独立业务配置区，全部输入框拦截{ } [ ] : " '等结构化符号，输入时实时弹窗拦截：
Agent 身份：单行纯文本输入框；
Agent 技能：点击「+」新增多条纯文本技能描述；
Agent 业务规则：点击「+」新增多条纯文本业务规则；
所有内容原样写入flow.yml，MCP 不解析、不执行业务逻辑。
6.2 节点回退编辑功能
Agent 表单左上角提供向左箭头按钮：
当前 Agent 已填写全部参数临时缓存不丢失；
页面切换至上一上游 Agent，支持修改上游配置；
多节点来回切换，全部配置缓存不丢失。
6.3 Agent 表单底部双固定按钮
【新增下个 Agent 节点】：缓存当前配置，新建下游空白 Agent 表单；
【工作流设置完成】：停止新增节点，弹出强制命名弹窗。
6.4 工作流强制命名规则
点击完成后弹窗强制输入自定义工作流名称，空值 / 非法字符禁止确认；
名称作为流程根目录文件夹名，写入flow.yml顶层；
提示文案：「必须设置工作流名称，后续离线终端需进入该目录执行启动指令」；
/xujin仅全局示例指令，高级模式可自定义全局离线启动别名。
7 Agent 门禁 + 单次执行哈希防伪 ID 机制（唯一性规则修正）
7.1 ID 签发规则（本轮执行全局唯一，定稿）
签发主体：离线本地后端门禁，不依赖 MCP；单份交付物基于完整文件内容哈希生成防伪 ID；
唯一性约束（更新）：同一次完整 / 断点执行链路内，全流程所有 Agent 产出的每一份交付物防伪 ID 全局永不重复；
同一 Agent 单次多份交付物 ID 互不重复；
不同 Agent 产出交付物 ID 同样禁止重复；
签发前读取本轮state_store已存在 ID 池查重，哈希冲突自动重新计算生成全新 ID；
生命周期：单次执行有效，新一轮/xujin启动自动清空state_store，上一轮全部防伪 ID 失效，仅管控本轮链路 Agent 流转状态。
7.2 门禁两层校验流程
合规校验层：文件名 / 格式与配置精准完全匹配、文件存在有效业务数据、三层文件校验全部通过；
ID 签发标记层：校验通过后生成全局唯一哈希防伪 ID 存入本轮状态库，标记 Agent 状态（completed_pass/completed_fail/pending/error）；校验失败不生成防伪 ID，阻断下游流转，日志留存拦截原因 + 文件路径。
7.3 跨 Agent 流转强制校验规则
向下游分支流转前，执行器逐份校验上游交付物：必须携带本轮执行内生成的全局唯一有效防伪 ID；任意一份交付物缺失合规 ID，直接阻断分支流转，日志标注缺失 ID 的文件路径。
7.4 分支判定规则
分支唯一判定依据：上游 Agent 最终执行状态字符串；
匹配逻辑：状态文本与配置精准完全匹配才命中对应分支，无模糊匹配；
标准状态枚举：completed_pass、completed_fail、pending、error；
业务示例：
tester Agent 门禁校验通过、状态 =completed_fail → 流转开发 Agent，绑定《测试执行清单》；
tester Agent 门禁校验通过、状态 =completed_pass → 流转验收 Agent，绑定《完整测试报告》；
隔离边界：插件仅管控流转路由、门禁、防伪 ID、交付物绑定，绝不干预任何 Agent 内部自有 Skill 逻辑。
8 断点 / 局部流程执行能力
8.1 三种执行模式（统一由离线/自定义指令承载）
完整从头执行整条流程；
断点续跑：从指定 Agent 名称向后执行后续链路；
局部片段执行：仅运行指定 Agent 之后的流程片段。
8.2 节点标识规则
仅使用Agent 名称作为唯一断点启动标识，最小执行粒度为单个完整 Agent。
指令示例：
bash
运行
# 完整从头执行，多文件批量传入
/xujin --start root --input file1.json,report.pdf
# 断点：从测试Agent开始执行
/xujin --start tester --input test_data.xlsx,config.yml
8.3 交付物上传约束
单次执行仅传入一组多文件上下文，一次性供给起始 Agent；
禁止分批次为多个节点单独上传独立文件；
单组上下文支持多份交付物同时传入。
8.4 入参三种传入方式
命令行逗号分隔多文件显式传参；
不传--input时交互式选择input_source历史文件；
终端直接拖拽文件自动拼接路径填充命令行。
9 新手 / 高级模式分层规则
9.1 模式持久存储
模式标记写入单工作流flow.yml，离线、重连 MCP 均保持对应视图与运行逻辑。
9.2 切换指令与视图表现
开启高级模式：mscp_workflow_advanced_switch enable
永久切换至完整高级视图，界面展示全部高级配置面板；
同步自动开启模板动态变量、审计日志档位 3、MCP 端故障人工介入面板；
切回新手模式：mscp_workflow_advanced_switch disable
隐藏全部高级配置面板；日志自动降级为标准极简日志档位 2；仅保留基础节点级重试；
默认状态：新建工作流初始为新手模式，不会自动开启高级视图。
9.3 模板分层联动
新手模式：仅基础模板下拉选择，隐藏变量编辑区域；
高级模式解锁：支持模板上下文动态变量占位符注入；配置持久存入flow.yml，离线可识别。
10 离线执行日志分层体系（与模式自动联动）
新手标准日志（档位 2 默认）：时间戳、上下游 Agent、本次起始 Agent、Agent 状态、交付物全局唯一防伪 ID、文件路径、故障拦截原因，按日期生成轻量 JSON 单行文件；
高级审计日志（档位 3，开启高级模式自动启用）：在标准字段基础上新增门禁校验详情、校验失败证据文件路径，仅存储文件路径不存储大段文本；
历史日志永久留存，无自动清理机制。
11 MCP 重连数据轻量化回读逻辑
仅读取本地flow.yml+ 日志文件，不依赖 MCP 全局global_config.json：
时间戳快速定位故障执行时段；
展示本轮起始 Agent、上下游流转 Agent、所有交付物全局唯一防伪 ID、各 Agent 最终状态；
清晰标注故障根因：门禁校验不通过 / 无有效业务数据 / 缺失本轮防伪 ID / 格式不匹配 / 自定义校验脚本拦截；
仅附带本地证据文件路径，用户自行打开核查原始交付物，内存占用极低；
历史执行日志内的防伪 ID 仅作复盘展示，不可在新一轮流程流转中复用。
12 异常阻断与重试分层策略
新手模式默认（档位 2）：节点级单独重启，完整保留上游所有文件、交付物、临时状态上下文；无可视化故障面板，仅日志记录故障信息，手动选择 Agent 重试；
高级模式解锁（档位 3）：节点重启 + MCP 可视化故障详情面板（仅在线连接 MCP 编辑时可见，离线无面板）；面板展示故障时间、上下游 Agent、本轮全局唯一防伪 ID、门禁拦截原因、证据文件路径；提供修改输入文件、重跑当前 Agent、终止全流程三类操作按钮；
离线运行无论高低级模式，仅提供基础节点重试功能，无可视化交互面板。
13 存量外部工作流导入转换规则
双导入入口：可视化界面上传目录 / ZIP 包 / 终端指令 mscp_workflow_convert --source 本地路径；
转换核心逻辑：MCP 仅生成标准目录骨架，交由大模型自主解析、填充全部配置，配套《存量工作流交互数据填充说明》作为映射标准，MCP 不干涉业务字段匹配；
格式清洗：原始 Agent 身份 / 技能 / 规则包含 JSON / 数组 / 结构化符号时，自动清洗为纯文本，界面弹窗标注提示用户核对；
转换完成强制输入自定义工作流名称，转换后完整保留用户原有 input_source/output_delivery 业务文件，不删除原始数据；
转换产物直接支持 MCP 可视化编辑、离线/自定义指令完整运行，快速交付闭环。
14 工作流导入导出方案（选定方案 2）
导出模板：MCP 专属指令mscp_workflow_export一键打包 ZIP，仅包含 flow.yml、templates、validate_rules、flow_fragments；排除运行生成的 input_source/output_delivery/logs/state_store，用于流程模板分享；
导入模板：上传 ZIP 包自动识别标准目录结构，加载完整流程配置；
完整备份：用户手动复制工作流根目录，留存全部历史输入、交付物、日志、版本快照。
15 MCP 指令分层交互体系（混合模式 3）
15.1 新手默认模式：拆分独立短指令（语义直白）
mscp_workflow_create 新建工作流
mscp_workflow_run 在线预览运行
mscp_workflow_export 导出流程模板
mscp_workflow_import 导入流程模板
mscp_workflow_convert 转换外部存量工作流
mscp_workflow_snapshot_create 生成版本快照
mscp_workflow_snapshot_rollback 快照回滚
mscp_workflow_disconnect 断开 MCP 切换离线运行
mscp_workflow_log 查看标准执行日志
15.2 高级模式统一复合主指令
开启高级模式指令：mscp_workflow_advanced_switch enable
解锁统一复合主指令 mscp_workflow_advanced，承载动态变量模板编辑、审计日志开关、全局 / 单流程黑白名单覆盖、故障人工面板、门禁哈希防伪参数配置、自定义校验脚本管理等全部高级复杂配置；
关闭指令 mscp_workflow_advanced_switch disable 切回新手视图，所有高级配置持久写入 flow.yml 不丢失。
16 离线终端执行指令体系（定稿）
全局离线启动指令自定义：/xujin为系统默认示例，高级模式可自定义全局别名；卸载 MCP 后指令依旧可用；
内置帮助指令格式：/自定义指令 -help，执行后终端打印当前工作流全部执行指令 + 逐条功能说明；
容错逻辑：
不在工作流根目录执行指令：提示未检测 flow.yml，引导进入对应流程目录；
--start传入不存在的 Agent 名称：输出当前流程全部合法 Agent 列表，终止执行；
文件不满足黑白名单基础校验：直接阻断并输出失败原因 + 文件路径；
新一轮执行自动清空 state_store 临时状态池，上一轮全部防伪 ID 永久失效；
流转校验缺失本轮全局唯一防伪 ID：拦截跨 Agent 流转，日志标注缺失文件路径。
17 终端能力边界
纯终端无可视化 Agent 编辑表单、无左箭头回退上一 Agent 快捷切换功能；如需新增、修改 Agent 身份 / 技能 / 规则，必须连接 MCP 打开可视化图形界面操作；终端仅用于流程执行、快照、导入导出、模式切换等运维操作。
