# 存量工作流填充说明

本文件用于指导将外部存量工作流迁移到 MCP Xujin 标准目录结构后的配置填充。

## 转换后的目录结构

转换工具 (`mscp_workflow_convert`) 会自动生成以下标准目录：

```
自定义工作流名称/
├─ flow.yml
├─ templates/
├─ validate_rules/
├─ flow_fragments/
├─ input_source/
├─ output_delivery/
├─ logs/
├─ state_store/
└─ versions/
```

原有 `input_source/`、`output_delivery/` 中的业务文件会被完整保留，不删除。

## 需要手动填充的内容

### 1. Agent 映射

打开 `flow.yml`，将原有工作流中的每个 Agent 映射为 `agents` 数组中的一个条目：

```yaml
agents:
  - name: original_agent_name
    identity: 原身份 / 角色描述（纯文本）
    skills:
      - 技能描述1
      - 技能描述2
    rules:
      - 业务规则1
      - 业务规则2
    deliverables:
      - filename: output.md
        template: "# Output\n"
    next_agents:
      - downstream_agent_name
    branch_conditions:
      completed_pass: downstream_agent_name
      completed_fail: another_agent_name
```

### 2. 交付物模板

将原交付物模板放入 `templates/` 目录，并在 `deliverables` 中引用文件名。

### 3. 校验规则

- 全局黑白名单与文件大小阈值：编辑 `flow.yml` 中 `global.whitelist` / `global.blacklist`
- 自定义校验脚本：放入 `validate_rules/` 并在 `global.validation.custom_script` 中指定路径

### 4. 分支流转

根据原工作流的状态机，填充每个 Agent 的 `branch_conditions`：

- `completed_pass`
- `completed_fail`
- `pending`
- `error`

分支匹配采用**精确字符串匹配**，无模糊匹配。

### 5. 模式与日志

- `mode`: `beginner` 或 `advanced`
- `log_level`: 新手模式 `2`，高级模式 `3`
- `offline_command`: 默认 `/xujin`，高级模式可自定义

## 清洗规则

若原始配置中包含 `{ } [ ] : " '` 等结构化符号，转换工具会提示清洗为纯文本。请在填充时再次核对：

- Agent 身份、技能、业务规则仅支持纯文本
- 禁止 JSON / 数组 / 大括号等结构化符号

## 验证

填充完成后，执行：

```bash
cd /path/to/workflow
/xujin --start root --input sample.txt
```

确认无校验错误且各 Agent 状态正确流转。
