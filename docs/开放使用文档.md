
文档基础信息
文档名称：MCP Xujin 工作流插件开放使用文档
面向人群：终端使用者、存量工作流迁移人员、运维人员
用途：操作指引、指令手册、存量导入迁移流程、离线运行完整说明
1 产品简介
本 MCP 插件为多 Agent 流程可视化编排工具，核心特性：
编辑依赖 MCP 服务，流程目录本地自包含，断开 MCP 后完全离线运行，无任何依赖；
职责边界清晰：仅管控文件校验、输入输出归档、门禁哈希防伪、Agent 状态分支流转；不修改、不校验、不限制任何 Agent 身份、技能、内部业务规则；
防伪 ID 规则：单次执行链路内全流程所有交付物哈希 ID 全局唯一，不同 Agent 之间 ID 不重复；仅本轮执行有效，重新执行自动失效；
支持完整运行、断点从指定 Agent 执行、局部片段运行；
分层视图：新手极简模式 / 高级完整模式，一键切换；
配套版本快照、存量外部工作流一键转换、导入导出模板、分层日志、节点级故障重试；
统一终端离线执行指令，内置 help 帮助文档，支持多文件批量入参、交互式选择、拖拽填充路径。
2 环境部署与启动说明
2.1 MCP 服务启动
支持 stdio 单编辑器独占、HTTP 多客户端并发两种连接模式，编辑器加载插件即可完成连接；
2.2 离线运行前置条件
已通过 MCP 可视化界面完成工作流编排，并填写自定义工作流名称；
终端进入该自定义名称对应的工作流根目录；
使用全局离线指令启动流程（默认示例 /xujin，高级模式可自定义全局指令别名）。
3 可视化界面完整操作步骤
3.1 新建标准工作流
编辑器执行新手指令 mscp_workflow_create，唤起可视化编辑界面；
Agent 节点配置流程：
1）填写 Agent 身份、技能、业务规则：输入框禁止{ } [ ]等结构化符号，系统实时拦截非法字符；点击「+」可新增多条技能、规则；
2）配置文件黑白名单、三层校验规则、交付物模板、分支流转条件；
3）左上角左箭头按钮：点击缓存当前参数，返回上一 Agent 修改配置，所有填写内容不丢失；
4）表单底部双按钮：【新增下个 Agent 节点】拓展链路 / 【工作流设置完成】结束节点编排；
点击【工作流设置完成】，弹窗强制输入自定义工作流名称，无名称无法生成流程目录；
编排完成后执行 mscp_workflow_disconnect 指令，切换离线独立运行模式。
3.2 存量外部工作流一键转换迁移
两种操作入口：
可视化界面：点击【导入外部工作流】，上传本地文件夹或 ZIP 压缩包；
终端高级指令：mscp_workflow_convert --source /本地工作流路径
转换执行逻辑：
1）系统自动生成标准完整目录结构，补齐 input_source/output_delivery/logs/state_store/versions；
2）交由内置大模型依据《存量工作流填充说明》自主解析原有 Agent、交付物、校验规则、流转链路；
3）原始配置含 JSON / 数组符号时，自动清洗为纯文本并弹窗提示人工核对；
转换结束填写自定义工作流名称，即可直接可视化编辑、离线执行；原有业务输入、交付物完整保留不删除。
3.3 版本快照管理操作
生成快照：mscp_workflow_snapshot_create，打包流程配置、模板、校验片段存入 versions 目录；快照不含运行产生的业务数据；
快照回滚：mscp_workflow_snapshot_rollback --snapshot 快照文件名.zip，一键覆盖当前流程配置；
清理说明：无自动清理、无一键清空快照指令，磁盘占用过高需手动进入 versions 目录删除文件。
3.4 新手 / 高级模式切换操作
开启完整高级视图（解锁全部高级功能）
plaintext
mscp_workflow_advanced_switch enable

生效效果：
界面展示全部高级配置面板；
自动开启模板动态变量、审计日志、MCP 故障人工介入面板；
解锁统一复合主指令 mscp_workflow_advanced 处理复杂配置；
切回新手极简视图
plaintext
mscp_workflow_advanced_switch disable

生效效果：隐藏高级配置，日志降级为标准极简日志，仅保留基础节点重试；
3. 模式状态永久写入 flow.yml，切换不丢失任何模板、快照、校验配置。
4 离线终端指令全手册（以默认示例指令 /xujin 为例）
4.1 帮助指令查看全部说明
bash
运行
/xujin -help

终端输出完整指令清单 + 释义：
plaintext
=== 【自定义工作流名称】离线执行帮助 ===
1. /xujin --start root --input file1.json,report.pdf
   作用：从头完整执行整条工作流，逗号分隔批量传入多份启动交付物
2. /xujin --start Agent名称 --input xxx.file
   作用：断点局部执行，从指定Agent向后运行，一次性传入起始Agent全部所需交付物
3. /xujin
   作用：交互式执行，自动读取input_source目录历史文件，数字多选传入
4. /xujin -help
   作用：打印当前工作流全套执行指令与说明
运行提示：输入文件、交付物、日志、临时防伪状态持久存储，无自动清理，请定期手动维护磁盘
约束说明：单次执行仅支持一组多文件输入，不可分批次给多个节点单独上传；跨Agent流转依赖本轮全局唯一哈希防伪ID，缺失直接阻断流转

4.2 通用执行指令示例
bash
运行
# 完整从头运行
/xujin --start root --input data.json,config.md
# 断点从测试Agent执行
/xujin --start tester --input test_result.xlsx
# 交互式选择历史文件启动
/xujin

4.3 三种入参填充方式
命令行手动逗号分隔多文件；
不传--input参数，交互式数字选择历史文件；
终端窗口直接拖拽本地文件，自动拼接完整路径填充命令行。
5 流转与防伪 ID 运行说明
Agent 完成执行提交合规交付物后，门禁校验通过才生成本轮全局唯一哈希防伪 ID；
同一执行链路内，所有 Agent 产出交付物 ID 互不重复；新一轮执行自动清空全部 ID，旧 ID 失效；
向下游 Agent 流转强制校验所有上游交付物携带本轮有效防伪 ID，缺失直接拦截并打印文件路径；
分支流转仅依据 Agent 执行状态精准完全匹配，无模糊匹配。
6 日志与故障排查
新手模式：输出标准极简日志，包含时间、Agent 名称、防伪 ID、故障文件路径；
高级模式自动开启审计日志，新增门禁校验详情、失败证据文件路径；
MCP 重连后可读取历史日志复盘每轮执行的起始 Agent、防伪 ID、流转状态，仅展示文件路径供用户自行核查原始数据。
7 导入导出模板备份操作
导出分享模板：mscp_workflow_export，打包 ZIP 仅含配置、模板、规则，不含运行业务数据；
导入外部标准流程模板：mscp_workflow_import，解压自动识别目录结构加载；
完整历史备份：手动复制整个工作流根目录，可留存全部输入、交付物、日志、版本快照。
8 常见限制说明
Agent 身份、技能、业务规则仅支持纯文本，禁止 JSON、数组、大括号等结构化符号；
修改 Agent 配置、回退上一节点仅可视化界面支持，纯终端无法编辑 Agent 表单；
离线运行无可视化故障面板，仅支持基础节点级重试；
全局配置 global_config.json 仅 MCP 编辑阶段读取，离线运行完全不依赖该文件；
所有运行产生的 input_source/output_delivery/logs/state_store 无自动清理机制，需用户手动维护磁盘。
